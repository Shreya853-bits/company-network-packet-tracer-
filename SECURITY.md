# Security Policy

This repository is a simulated lab. It contains lab-only usernames, passwords, and IP addresses.

## Lab Credentials

The configs include simple credentials such as:

- `admin`
- `Admin123!`
- `class`
- `cisco`

These are intentionally weak because they are for Packet Tracer practice only. Do not reuse them on real devices.

## Reporting Issues

Open an issue if you find:

- A security control that does not match the documented policy.
- An ACL rule that allows traffic the lab says should be blocked.
- A credential or address that appears to come from a real environment.

## Scope

In scope:

- Router and switch config correctness
- ACL behavior
- NAT behavior
- Port security behavior
- Documentation accuracy

Out of scope:

- Real Cisco device hardening
- Production network security
- Real incident response
